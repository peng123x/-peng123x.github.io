前言

大家好 去年一直在学习Unity3D， 之后发现了图形学，就想往图形方面发展， 刚开始的时候一直在无头苍蝇一般的研究一些渲染管道，学习Opengl 自己还做了一些Demo，我的目标是写一个属于自己的mimi渲染引擎 虽然自己写程序方面能力很弱 慢慢来，鲁迅曾经说过 以教为学 也就是把教别人的过程作为帮助自己学习的过程，也就是把教别人的过程作为帮助自己学习的过程。所以本人开创了属于自己的博客 可能还是在朋友圈分享一下学习录和分享一下自己学习的过程(以后不知道)。 上半年一直在研究Opengl API 虽然是一个很老的API，给我提供了非常宝贵的学习经验，而且我个人也在使用，准备用他写我上面提到的属于自己的渲染引擎。世界一直在变化，学习的知识不断的创新，老的东西在虚拟网世界毕竟是要淘汰的，所以我开始了Directx12 的学习，以前对Directx 11也一些了解和看了相关方面的书(虽然没看完哈哈哈)，他是全新的API(一直还在完善)，只在Win10系统上提供支持，所以他是一些计算机图形等方面重要部分的未来，值得学习。之后我也了解了Vulkan，也是Opengl的代替者，完善了一些Opengl的Bug，毕竟人的精力有限，我也在学习别的方面的知识，所以我的博客还是围绕着Directx转，我也不是什么大佬 哈哈哈。

上面描述了那么多，写博客也可以促进我的文学和表达方面的知识，毕竟教别人的同时也是强化自己的一个过程，最主要还是写给自己看 哈哈哈。小时候出国的时候，中文水平也不是那么好，到了国外就开始学习西班牙语，写的作文也是围绕的西语转，可能自己中文表达能力不是很强，通过写博客的交流过程可能让自己对语言表达方面更加完美吧。上几个月在实习的时候，虽然和他们部门实习的过程很坎坷(部门经理不会写程序，难交流 etc)，她让我学到了怎么好好的写报告，那段时间程序没多写，报告写了100多页。 她让我认识到了要完善自己的写作能力和解释能力，毕竟你写的只有你自己看得没什么用，只有好的表达才是最好的交流，以后工作你也是要和同事交流的，写程序也是要写文案。所以这段经历吧，可以让我好好的成长

小时候一直喜欢做游戏，也对做游戏没什么概念，到大学了以为学计算机工程学可以学到着方面的知识，还是自己太年轻了哈哈哈哈。到了大三才开始慢慢了解这方面的知识，那段时间一直被作业压着，没精力去学这些东西。到了大四才开始有时间学习，深入了解开发游戏的过程后，就喜欢上了图形学，发现了新的大陆。原来计算机渲染图形是那么的深奥，你不但要了解他的理论，还需要一些物理知识，什么基于物理的渲染，屏幕空间环境光遮蔽等，每一项都是包含很多要学习的地方的。所以现在一直想了解他们的原理，比做游戏还更加感兴趣。虽然路程很艰辛，我也不是什么学霸，也没他们那个天赋，我会坚持下去的。

我网址是在 github 创建的，网页也是网上的模板下载过来，毕竟人生苦短 我甩模板，切克闹。 以后写想发表一些文案或者程序，我会先写在Markdown上，然后用python转化html ，再把那段文字贴在模板上，省去我很多时间。以后一些文章还是可能用英文写吧，毕竟计算机中很多概念术语最初都是英文命名，代码也按照英文来定义关键字。那如果是汉语 计算机就崩溃了 比如说 如果 （杯子里 有水） 就 喝口水 那么这个喝口水 计算机怎么解释？是（（喝口）水）还是 （喝 （口水））
